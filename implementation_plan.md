# Goal Description

The goal of this task is to develop a premium, separate **Admin Project** under `project/admin` that interacts with the same database as the `project/user` platform.

The Admin Project serves three major administration profiles:
1. **Event Manager**:
   - Manage Events: Add, modify, and cancel events. We will add `status` and `banner_url` to `events`.
   - Manage Venues: Add/modify venues and capacity.
   - Schedules: Choose an event and a venue, and configure multiple `event_days` (`date`).
2. **Ticketing Staff**:
   - Configure ticket prices, types, and max quantities via `ticket_configs` for a schedule.
   - **Visual Seat Grid Editor**: Arrange physical seats in a grid (A-J rows, up to 18 columns, matching the layout in `sample_map.png`) where seats can be STANDARD, VIP, or COUPLE (which occupies 2 columns).
   - Capacity Validation: Ensure the layout's total seat count does not exceed the venue's overall capacity.
   - **Pre-generated e_tickets**: When ticket configurations/seat layouts are saved for a schedule, the system automatically pre-generates the `e_tickets` list representing physical seats (`row_label` and `col_number`) for *each event_day* of this schedule!
3. **Artist Coordinator**:
   - CRUD for Artists.
   - Assignment: Assign main and backup artists for each `event_day` with validation enforcing **at least 2 backup artists** per day.

We are adhering strictly to the database design in `report/schema.png` and the PDF:
- The legacy `booking_details` table is removed; `e_tickets` now references `bookings` directly.
- The `e_tickets` table stores seat coordinates directly via `row_label` and `col_number`.
- No additional seats/day-seats tables are created.

  - `artist_name` (varchar NN)

  - `config_id` (integer PRIMARY KEY)
  - `schedule_id` (integer REFERENCES event_schedules)

  - `booking_id` (integer PRIMARY KEY)

### Database Configurations

We will apply the DB adjustments using a raw SQL patch script.

#### [NEW] [schema_patch.sql](file:///c:/yuuhi/courses/3rd%20year%20-%202nd%20sem/INT2020%201%20-%20PTTK/project/admin/schema_patch.sql)
- Raw SQL script to drop booking_details table, adjust e_tickets columns, and add status/banner columns.
  ```sql
  -- Drop booking_details table and dependent FKs if they exist to match schema.png
  DROP TABLE IF EXISTS public.e_tickets CASCADE;
  DROP TABLE IF EXISTS public.booking_details CASCADE;

  -- Add status & banner columns to events
  ALTER TABLE public.events ADD COLUMN IF NOT EXISTS status character varying(50) DEFAULT 'ACTIVE';
  ALTER TABLE public.events ADD COLUMN IF NOT EXISTS banner_url text;

  -- Add registration_start & registration_end to event_schedules to match schema.png
  ALTER TABLE public.event_schedules ADD COLUMN IF NOT EXISTS registration_start timestamp without time zone;
  ALTER TABLE public.event_schedules ADD COLUMN IF NOT EXISTS registration_end timestamp without time zone;

  -- Create e_tickets exactly matching schema.png + seat extensions (row_label & col_number)
  CREATE TABLE public.e_tickets (
      ticket_id integer NOT NULL GENERATED BY DEFAULT AS IDENTITY,
      ticket_config_id integer NOT NULL,
      event_day_id integer NOT NULL,
      booking_id integer,
      ticket_code character varying,
      ticket_status character varying DEFAULT 'Available' NOT NULL, -- 'Available', 'Holding', 'Valid', 'Used', 'Canceled'
      row_label character varying,
      col_number integer,
      CONSTRAINT e_tickets_pkey PRIMARY KEY (ticket_id),
      CONSTRAINT e_tickets_ticket_config_id_fkey FOREIGN KEY (ticket_config_id) REFERENCES public.ticket_configs(config_id) ON DELETE CASCADE DEFERRABLE,
      CONSTRAINT e_tickets_event_day_id_fkey FOREIGN KEY (event_day_id) REFERENCES public.event_days(event_day_id) ON DELETE CASCADE DEFERRABLE,
      CONSTRAINT e_tickets_booking_id_fkey FOREIGN KEY (booking_id) REFERENCES public.bookings(booking_id) ON DELETE SET NULL DEFERRABLE
  );
  ```
